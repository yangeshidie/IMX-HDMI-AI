# 02_mutex_lock - 数据竞争与互斥锁 (Mutex) 性能实验

本目录包含多线程竞争共享资源与互斥锁 (`pthread_mutex_t`) 的对照实验，直观展示竞态条件 (Race Condition) 的危害以及不同加锁粒度对程序性能的影响。

---

## 文件列表

- [main.c](./main.c) : 对照测试共享变量递增（无锁 vs 循环内频繁加锁 vs 循环外临界区加锁）的耗时与计算准确性。
- [Makefile](./Makefile) : 本地编译构建脚本。

---

## 对照实验总结

测试两个线程并发对全局变量 `gCount` 进行 100 万次 `++` 操作：

| 实验组 | 加锁策略 | 数据准确性 | 耗时性能表现 | 原因分析 |
| :--- | :--- | :--- | :--- | :--- |
| **无锁组** | 不加锁 | 结果异常 (小于200万) | 极快 | 存在数据竞争 (Data Race)，写覆盖导致计数值丢失。 |
| **循环内加锁** | 每次 `++` 频繁加锁解锁 | 结果正确 (精确200万) | 较慢 | 线程在循环内频繁上下文切换与竞争锁，系统开销大。 |
| **循环外加锁** | 进函数直接锁死，计算完再开锁 | 结果正确 (精确200万) | 极快 | 极大减少了锁竞争频率，保护了整段临界区。 |
