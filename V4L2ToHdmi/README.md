# V4L2ToHdmi - 文件说明

这个文件夹下包含了从 V4L2 摄像头采集视频到 DRM 显示出图的一系列渐进式测试代码。下面是各个 C 代码文件及其 Makefile 的功能说明：

### 1. `camera_test.c`
**基础设备探查程序**：用于探测摄像头设备节点（如 `/dev/video11`）的格式能力。它会尝试查询单平面（`VIDEO_CAPTURE`）以及多平面（`VIDEO_CAPTURE_MPLANE`）模式下的长宽分辨率信息，验证摄像头驱动是否能被正常访问。

### 2. `drm_green.c`
**DRM 基础显示框架测试**：仅关注 DRM（Direct Rendering Manager）显示端的操作。程序会申请显存（Dumb Buffer）、映射内存，并通过 CPU 填充纯绿色数据推送到 HDMI 屏幕上，停留 5 秒后恢复原始屏幕状态，用于验证屏幕显示通路是否正常。

### 3. `drm_Aframe.c`
**V4L2 + DRM 单帧图像联调**：V4L2 采集与 DRM 显示的基础联动代码。采集一帧 V4L2 摄像头图像（NV12 格式），完全使用 **CPU** 将其进行格式转换和尺寸缩放（NV12 转 XRGB8888），随后推送到 DRM 渲染上屏显示，直至按下回车退出。

### 4. `drm_Aframe_RGA.c`
**CPU 与 RGA 硬件加速单帧性能对比**：在获取单帧图像并渲染的基础上，引入了瑞芯微的 RGA（硬件 2D 加速）模块。该文件会分别采用 CPU 与 RGA 完成同样的图像格式转换和缩放任务，并在控制台输出精准计算的毫秒级耗时对比，验证硬件加速带来的性能提升。

### 5. `final_video_test.c`
**综合连续视频流与极限压测程序**：项目的最终演进版本。内置两种测试模型，支持通过命令行参数（如传 `0`、`1`、`2`）切换：
- **实时视频流测试 (Streaming Test)**：循环获取摄像头视频流并在屏幕实时显示，输出并对比连续渲染时的 CPU 与 RGA 流畅度与帧率表现。
- **极限性能压测 (Limit Benchmark)**：锁定缓存中的单帧图像连续不停地交给 CPU/RGA 计算，用于测试得出极端纯转换场景下的理论最高帧率（FPS）及最小处理延迟。

### 6. `Makefile`
**工程构建与自动部署配置**：用于定义各个程序的 `aarch64` 架构交叉编译规则，链接了后续所需的 `libdrm` 与 `librga` 动态库。此外还提供了 `make deploy` 和 `make run` 指令，能够自动将最新编译的二进制文件通过 SCP/SSH 传输推送到配置的目标开发板（IP: `10.225.235.41`）上直接运行，提升联调开发效率。
