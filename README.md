# IMX-HDMI-AI

基于RK3576构建嵌入式视觉系统，打通IMX415（V4L2）→DRM/KMS→HDMI显示链路；引入DMA-BUF 实现零拷贝数据传输，将端到端延迟控制在120ms内。 硬件加速与流媒体处理：基于MPP实现H.264硬件编码，满足实时访问需求。采集/编码/AI解耦的多线程架构，实现实时推理与事件触发机制，支持关键视频片段自动保存与标记。

# V1.2

完成编码测试，准备编写推流。

## 环境说明
  - 开发板：**立创·泰山派3M-RK3576**
  - 板上系统：**Debian**
  - 屏幕：**1024x600HDMI屏幕**（非标准分辨率，需进行HDMI 物理层驱动适配）
  - 摄像头：**IMX415**（4k分辨率、MIPI接口）
  - 虚拟机操作系统：**Ubuntu 22.04.5 LTS**

## 目录说明
  - V4L2ToHdmi：摄像头与HDMI屏幕配置，通过V4L2实现摄像头获取帧零拷贝通过RGA转化后显示在HDMI屏幕上，并内置了多种测试。
  - MultiThread：Linux下多线程多线程学习路线（简要）。最后将使用生产者-消费者模型将摄像头-HDMI拆解为两个线程。并且重写V4L2ToHdmi目录下耦合在一起的代码，进行模块化。
  - VideoEncoderAndPush：添加硬件编码和推流功能，并且微调架构，引入生命周期管理，采用生产者分发帧的形式。
